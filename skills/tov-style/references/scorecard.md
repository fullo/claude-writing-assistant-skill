# Scorecard di Qualità — Capitolo

Assegnare un punteggio 0–10 per ogni dimensione. Obiettivo: ≥ 60/70 per ogni capitolo.

| # | Dimensione | Punteggio | Cosa verificare |
| :--- | :--- | :---: | :--- |
| 1 | Struttura narrativa | __/10 | Hook con caso studio, pattern Esperienza → Problema → Soluzione → Esercizio |
| 2 | Zero ripetizioni | __/10 | Nessun concetto rispiegato; cross-reference corretti |
| 3 | Rapporto teoria/pratica | __/10 | ≤30% teoria, ≥70% pratica |
| 4 | Framework centrale | __/10 | Presente dove rilevante; linguaggio coerente |
| 5 | Tono | __/10 | Conversazionale-esperto; prosa non elenchi; mai presupponente |
| 6 | Linguaggio | __/10 | Sentence case; no inglesismi; termini di dominio corretti |
| 7 | Formattazione | __/10 | Conforme a Guida Markua; callout con label corrette |
| | **TOTALE** | **__/70** | |

## Checklist finale — Libro completo

### Struttura
- [ ] Tutte le Parti completate
- [ ] Tutti i capitoli con struttura obbligatoria (epigrafe, hook, Quick Win, Checkpoint)
- [ ] Esercizi pratici distribuiti nei capitoli (4–6 per libro)
- [ ] Appendici standard: Glossario, Template, Framework, Bibliografia, Caso studio completo

### Continuità narrativa
- [ ] Timeline del caso studio coerente tra tutti i capitoli
- [ ] Numeri (metriche, headcount) verificati contro timeline master
- [ ] Personaggi mantengono caratterizzazione consistente
- [ ] Cross-reference tra capitoli corretti e bidirezionali

### Qualità per capitolo
- [ ] Tutti i capitoli ≥ 60/70 in Scorecard
- [ ] Rapporto 30/70 teoria/pratica rispettato
- [ ] Framework centrale presente nei capitoli chiave
- [ ] Zero ripetizioni; cross-reference al posto di rispiegazioni
- [ ] Prosa narrativa; elenchi puntati solo dove ammessi
- [ ] Sentence case su tutti i titoli
- [ ] Conformità Markua verificata

### Serie "the Right Way"
- [ ] Riferimenti ad altri volumi come approfondimento opzionale (mai prerequisito)
- [ ] Terminologia coerente tra volumi dove i concetti si sovrappongono
- [ ] Callout senza emoji in tutti i libri; label standard

## Validazione stilistica italiana

Dopo la scorecard, eseguire la validazione con le 15 regole di `references/italian-rules.md`.

Formato di output:

```
[R#] Riga/passaggio: "testo originale"
  → Correzione: "testo corretto"
```

```
PUNTEGGIO DI CONFORMITÀ: __/100
REGOLE PIÙ VIOLATE: R__, R__, R__
RIEPILOGO: __ violazioni trovate su __ regole diverse.
```

Scala: 95–100 eccellente, 85–94 buono, 70–84 sufficiente, 50–69 problematico, <50 da riscrivere.
